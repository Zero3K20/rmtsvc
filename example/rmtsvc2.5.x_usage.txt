rmtsvc2.5.x Usage Guide

Program Overview:
1. Remote host monitoring/surveillance
2. Remote file/service/registry/process/telnet management
3. Feature-rich FTP service
4. Feature-rich proxy service, supporting secondary proxy, HTTPS/SOCKS4/SOCKS5
4. Complete vIDC management configuration, supporting local port mapping, remote port mapping, intranet proxy, proxy service, etc.
5. SSL VPN functionality, providing secure remote access protection for your intranet applications
6. msnbot functionality, allows managing the above features via MSN chat
7. UPnP support, directly perform port mapping on UPnP-capable gateways or routers (new, added 20070806)

[Running the Program]:
rmtsvc is a console application that can run as a system service. Supported command-line parameters:
-i [DisplayName] [ServiceDesc] --- Install as service
Example: rmtsvc.exe -i
         rmtsvc.exe -i test "Test Service"
-u --- Uninstall service
Example: rmtsvc.exe -u
-s --- Start service
Example: rmtsvc.exe -s
-e --- Stop service
Example: rmtsvc.exe -e
-d --- Run in debug mode; -d0 ~ -d4 specify different debug message levels; closing the console window will end the program
Example: rmtsvc.exe -d4
-f filename --- Specify the configuration file name; if not specified, defaults to a file with the same name as the program but with .ini extension
Example: rmtsvc.exe -d4 -f aa.ini
-c saveasfile --- Write current ini configuration parameters into the exe file itself and save as the specified file
-C saveasfile --- Write current ini configuration parameters into the exe file itself and save as the specified file; EXEs saved with this parameter will not support ini config files at runtime
Note: -c/-C parameters must be used together with -d
Examples: rmtsvc.exe -d4 -c rmtsvc1.exe
          rmtsvc.exe -d4 -c rmtsvc1.exe -f aaa.ini
          rmtsvc.exe -d4 -C rmtsvc2.exe


[About Configuration Files]:
rmtsvc supports ini configuration files. Users can specify the config file name with the -f parameter. If not specified, the program defaults to reading a config file with the same name as the program but with the .ini extension.
For example, if the program is rmtsvc.exe, the default config file is rmtsvc.ini.
The program reads configuration info in this order: first the configuration commands embedded in the exe, then the ini config file, then special command-line parameters.
For exe files saved with the -C parameter, the program will not read ini config files, even if an ini config file is specified at runtime.
The ini config file consists of a series of configuration commands supported by the program, one command per line.
If a line begins with !, it is a comment and will not be interpreted. See below for supported configuration commands.

[ini Configuration Commands]:
Note: In the command descriptions below, <> represents user-supplied content, [] means the item is optional, and | means one of the options can be entered.
If the value on the right side of = contains spaces, enclose the value with " characters. Example: sets log="c:\program files\aaa.log"

Set service information
Command format: 
sets [spyself=FALSE] [stop_pswd=<stop password>] [log=<log file>] [opentype=APPEND] [loglevel=DEBUG|INFO|WARN|ERROR]
spyself=FALSE      : Disable crash-restart monitoring. By default, crash-restart monitoring is enabled.
stop_pswd=<stop password> : Set the password to stop the service. If set, the service cannot be stopped without the password.
If set, the user can only stop the service via command line: -e <password>. Cannot stop via SCM or net stop.
For example, if the program name is xx.exe and stop password is 123: xx.exe -e 123
log=<log output file> : Set whether the program outputs a log file.
opentype=APPEND    : Set whether the program appends to or overwrites the log file on startup. Default is overwrite.
loglevel=DEBUG|INFO|WARN|ERROR : Set the log output level. Default is INFO.
faceless=TRUE : If started in non-service mode and -d is not specified, run the program without a window.
For example, when double-clicking to run the program, if this is set, it runs without a window.
Otherwise, run with a window; pressing Ctrl+C or closing the window will end the program.
install=TRUE : If set, the program will automatically install as a service on startup.
Example: sets stop_pswd=1234
 
Set web service information
Command format: 
webs [port=<web service port>] [bindip=<local IP>] [ssl_enabled=true] [ssl_verify=true]
port=<service port>    : Set the service port. Default is 7778. 0 means do not start web service; <0 assigns randomly.
bindip=<local IP> : Set the local IP to bind. If not set, binds to all local IPs.
ssl_enabled=true: Indicates SSL-encrypted service. If not configured, plain service.
ssl_verify=true: Only valid with ssl_enabled=true; requires client certificate verification. Default: no verification.

Example: webs port=7779 


Set visitor account permissions
Command format:
user account=<account> [pswd=<password>] access=<permissions>
account=<account> : Required. The rmtsvc visitor account to add.
pswd=<password>   : Required. The password for the specified account.
access=<permissions>
Defaults to ACCESS_NONE if not set. Format and meaning:
ACCESS_ALL=ACCESS_SCREEN_ALL|ACCESS_FILE_ALL|ACCESS_REGIST_ALL|ACCESS_SERVICE_ALL|ACCESS_TELNET_ALL|ACCESS_FTP_ADMIN|ACCESS_VIDC_ADMIN
ACCESS_SCREEN_ALL: Full remote screen control permissions
ACCESS_SCREEN_VIEW: Remote screen view-only permissions
ACCESS_FILE_ALL  : Remote file management (read/write/delete, etc.)
ACCESS_FILE_VIEW : Remote file management, read and download only
ACCESS_REGIST_ALL: Remote registry management (read/write/add/delete, etc.)
ACCESS_REGIST_VIEW: Registry view-only
ACCESS_SERVICE_ALL: Remote service management, full control
ACCESS_SERVICE_VIEW: Remote service, view-only
ACCESS_TELNET_ALL:  Remote telnet management
ACCESS_FTP_ADMIN : Remote FTP service configuration management
ACCESS_VIDC_ADMIN : Remote vIDC management configuration
! Multiple accounts can be configured, one per line

Example: user account=aa pswd=bb access=ACCESS_ALL

Set MSN bot information
Command format:
msnbot account=<msn account>:<password> [trusted=<trusted msn account>] [prefix="<mobile MSN reply prefix>"]
account=<msn account>:<password> : Specify the login account and password for the MSN bot.
trusted=<trusted msn account>: Specify an MSN account trusted by the MSN bot.
  This account can control the MSN bot without entering the access password.
  Multiple accounts can be entered, separated by ','.
prefix="<mobile MSN reply prefix>" : Prefix to strip from incoming messages before interpretation.
Example: msnbot account=xxx@hotmail.com:aaa prefix="The other party is using mobile MSN, see http://mobile.msn.com.cn."

Note: If prefix is not configured, the default prefix is "The other party is using mobile MSN, see http://mobile.msn.com.cn."
Note: If an MSN bot account is configured, it is also the rmtsvc access account with ACCESS_ALL permissions.

Example: msnbot account=XXX@163.com:111 trusted=aaa@hotmail.com

Configure MSN bot proxy
Command format: 
proxy type=HTTPS|SOCKS4|SOCKS5 host=<proxy address> port=<proxy port> [user=<account>] [pswd=<password>]
type=HTTPS|SOCKS4|SOCKS5 : Specify proxy protocol type
host=<proxy address>      : Specify proxy server host address
port=<proxy port>         : Specify proxy service port


rmtsvc.ini Example
sets
msnbot account=XXX@163.com:111 trusted=aaa@hotmail.com
webs port=7779
user account=aa pswd=1234 access=ACCESS_ALL


//------------------------------------------------------------------------------------------------------------------------
//---------------------------------------------------------MSN Bot Supported Commands------------------------------------
//===========All commands below can be entered in the MSN bot chat control window
//===========Prefix each command with #
//===========Example: to terminate process aa.exe, enter: #terminate aa.exe
terminate - Terminate the specified process
   Usage: terminate <process name>,<process name>,<process name>...

exec - Launch the specified program
   Usage: exec [-hide] [-check <programname>] [-user <[Domain\]account:password>] [-wait <dwMilliseconds>] <full path program>
   -hide  - Run the specified program in the background
   -check <programname> - Check if <programname> is already running before executing (case-insensitive); if so, do not execute.
   -user <[Domain\]account:password> - Start program with specified account; only valid when running as a service.
   -wait <dwMilliseconds> - Wait behavior: ==0 don't wait; <0 wait until process ends; >0 wait specified milliseconds.
   <full path program>  - Program and its parameters.

plst - List current processes on this machine
   Usage: plst [<filter>,<filter>,...]
       <filter> - Specify processes to list; supports * and ? wildcards. If omitted, list all.

path - Set the default storage path for uploaded files.
   Usage: path [<default storage path>]
If not set, return current path. If set to ., use msnbot program directory.

down - Download a file from a URL to the msnbot machine, or download from msnbot to local machine
   Usage: down <http://url>|<ftp://url>|<msnbot local file>

head - Specify the avatar for the msnbot
   Usage: head [<msnbot local image file>]

nick - Specify the nickname for the msnbot
   Usage: nick <nickname>

snap - Capture the current screen
   Usage: snap [<jpeg quality>]  (default 60; higher = better quality but larger file)

stat - Display system information (machine name, OS, CPU, workstation status)

keys - Simulate sending specified keystrokes/strings
    Usage: keys <string to send>

fpas - Find login account password
    Usage: fpas [<Domain>\<account>]

shutdown - Shutdown the machine
restart  - Restart the machine
logoff   - Log off the current user
lock     - Lock the current machine (NT/2K/XP only)
3key     - Simulate Ctrl+Alt+Del
  
vcap - Camera commands
   vcap status - View camera information
   vcap snap [<jpeg quality>] - Capture camera image
   vcap sets [size=<width:height>] [device=<device index>] - Select camera or set image size

www - Start or stop rmtsvc web remote control service
   www stop / www start [<port>] / www (status)
  
ftp - Start or stop FTP service
   ftp stop / ftp start [<port>] / ftp (status)

proxy - Start or stop proxy service
   proxy stop / proxy start [<port>] / proxy (status)
   
vidcs - Start or stop vidc service
   vidcs stop / vidcs start [<port>] / vidcs (status)

mtcpr - Perform remote port mapping
   Format: mtcpr vidcs=<host:port@pswd> appsvr=<host:port> mport=<map port>[+-ssl]
   Example: mtcpr vidcs=222.33.11.xx:8088@aa appsvr=192.168.1.11:21 mport=222
   
export - Export all exe/ftp/proxy/vidc configuration parameters
fport - Process/port association list
help  - List all supported commands

import - Import FTP/Proxy/vidc configuration parameters
Format: #import [ftp|proxy|vidc]<[config file path]
Example: #import ftp<c:\1.txt

upnp - UPnP device discovery and port mapping
    upnp find / upnp stop / upnp status
    upnp add type=[TCP|UDP] appsvr=<service> mport=<port> appdesc=<desc>
    upnp delete <port> <TCP|UDP>

update - Upgrade rmtsvc
Format: #update <new exe path or URL>

telnet - Start/stop telnet service, reverse telnet connection
   telnet stop / telnet start [<port>] [account=<account:password>]
   telnet -r <host>:<port>: Reverse-connect to get a CMD shell
   telnet (status)
Note: Supports same commands as msnbot (except MSN-specific: head, nick, snap, vcap).
     ini config: telnet [port=<port>] [bindip=<local IP>] [account=<account:password>]

2007-03-12
All msnbot commands support output redirection with >:
Example: #plst > a.txt
Then: #down a.txt

Q: Why does stopping a service via msnbot and then restarting it always fail?
A: Close the chat session window with the msnbot, reopen it, then enter the start command.
  The cmdshell inherits handles from its parent process, so the port is not released until the cmdshell ends.
